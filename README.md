# J-- Compiler (Haskell)

A compiler for **J--**, a simplified imperative programming language (a subset of Java-like syntax), written in Haskell. Developed for the Compilers course, Computer Science program at UDESC (Universidade do Estado de Santa Catarina).

The compiler performs lexical, syntactic, and semantic analysis, then generates **JVM bytecode** via Jasmin assembly, allowing J-- programs to actually run on the Java Virtual Machine.

## Pipeline

```
Source (.j--)
      │
      ▼
Lexer (Alex)         →  Lex.x
      │
      ▼
Parser (Happy)        →  Parser.y   (builds the AST)
      │
      ▼
Semantic Analysis     →  Semantico.hs  (type checking, scope, error reporting)
      │
      ▼
Code Generation        →  Gerador.hs   (emits Jasmin assembly, .j file)
      │
      ▼
Jasmin Assembler      →  .class file (JVM bytecode)
      │
      ▼
JVM                    →  java <ClassName>
```

- **Alex**: generates the lexer from `Lex.x`, tokenizing the source code.
- **Happy**: generates the parser from `Parser.y`, building the Abstract Syntax Tree (`AST.hs`).
- **Semantic analysis** (`Semantico.hs`): validates types, scoping, and variable declarations, reporting errors before code generation.
- **Code generation** (`Gerador.hs`): traverses the validated AST and emits Jasmin assembly instructions, which are then assembled into JVM bytecode.

## Requirements

- [GHC](https://www.haskell.org/ghc/) (Glasgow Haskell Compiler)
- [Alex](https://hackage.haskell.org/package/alex) (lexer generator)
- [Happy](https://hackage.haskell.org/package/happy) (parser generator)
- Java Runtime Environment (to run the generated bytecode)
- [Jasmin](http://jasmin.sourceforge.net/) (`jasmin.jar`, included in this repository, used to assemble the generated `.j` file into a `.class` file)

Install Alex and Happy via Cabal:
```bash
cabal install alex happy
```

## Building

```bash
alex Lex.x
happy --ghc Parser.y
ghc Main.hs
```

## Running

```bash
./Main <source_file.j-->
```

This generates a `Teste.j` file containing Jasmin assembly. To assemble and run it on the JVM:

```bash
java -jar jasmin.jar Teste.j
java Teste
```

## Example

An example J-- source file (`teste.j--`) is included in this repository to test the full pipeline.

## Repository Structure

```
Lex.x        # Lexer specification (Alex)
Parser.y     # Grammar and parser specification (Happy)
AST.hs       # Abstract Syntax Tree data types
Semantico.hs # Semantic analysis (type checking, scoping)
Gerador.hs   # JVM bytecode (Jasmin) code generation
Main.hs      # Entry point: wires the full pipeline together
teste.j--    # Example J-- source file
jasmin.jar   # Jasmin assembler, used to assemble the generated .j file
```
